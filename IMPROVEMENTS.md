# 改善ロードマップ

連想絵文字検索をブランチ分けしながら育てるための改善メモです。

## P1: 関連理由の表示

候補カードに、なぜ関連として出たのかを表示する。

例:

```text
🎂 誕生日ケーキ
共通: お祝い / 楽しい
```

狙い:

- 右側の関連候補に納得感が出る
- タグ調整時に、どのタグが強すぎるか見つけやすい

ブランチ候補:

```text
feature/related-reasons
```

## P1: タグクリックの見た目改善

中央のタグ、場面、トーンがクリック可能だと分かるようにする。

改善案:

- ホバー時に色を変える
- 選択中タグの見た目を強くする
- タグチップに小さなフィルター感を出す

狙い:

- 「右側を絞り込める」ことに気づきやすくする
- 現在どのタグで絞っているかを迷わないようにする

ブランチ候補:

```text
feature/tag-filter-visuals
```

## P2: 関連タブ

右側の関連欄を、関連の種類で切り替えられるようにする。

タブ案:

```text
すべて / タグ / 場面 / トーン / 同カテゴリ
```

狙い:

- 関連候補が多いときに探しやすくする
- 「意味が近い」「使う場面が近い」「雰囲気が近い」を分けられる

ブランチ候補:

```text
feature/related-tabs
```

## P2: 複数タグフィルター

右側の関連欄を、複数条件で絞り込めるようにする。

例:

```text
夏 + 食べ物
かわいい + 動物
謝罪 + 丁寧
```

狙い:

- 候補が多いときに段階的に絞り込める
- 連想検索らしい探索体験に近づく

注意:

- 最初から複雑にしすぎるとUIが重くなる
- まずはAND条件だけでよい

ブランチ候補:

```text
feature/multi-tag-filter
```

## P2: 初期表示の改善

検索語が空のときに全件を並べるのではなく、探し始めやすい表示にする。

案:

- 人気タグ
- 最近コピーした絵文字
- カテゴリ別の代表絵文字
- 季節・感情・場面の入口

狙い:

- 空画面や大量表示を避ける
- ユーザーが検索語を思いつかないときも使える

ブランチ候補:

```text
feature/home-discovery
```

## P3: 検索スコアの調整

検索結果の順位を調整しやすくする。

改善案:

- `name_ja` 完全一致を最優先
- `tags_ja` を `scenes_ja` より強くする
- `tone_ja` は少し弱める
- CSV側に重み列を追加する

狙い:

- 「夏」で検索したときに夏の代表絵文字が上に来る
- 汎用タグが強すぎて結果が散らばる問題を抑える

ブランチ候補:

```text
feature/search-scoring
```

## P3: 同義語辞書

検索語の表記ゆれや言い換えを吸収する。

例:

```json
{
  "嬉しい": ["笑顔", "楽しい", "喜び", "ハッピー"],
  "謝る": ["謝罪", "ごめん", "すみません", "お願い"],
  "夏": ["暑い", "海", "太陽", "花火", "スイカ"]
}
```

狙い:

- CSVの各絵文字にタグを増やしすぎずに検索性を上げる
- 連想検索として自然な入力を拾いやすくする

ブランチ候補:

```text
feature/synonyms
```

## P3: 公式データ取り込み

Unicode公式データからベースCSVを生成する。

候補データ:

- `emoji-test.txt`
- CLDRの日本語名称データ

狙い:

- 仮データから全絵文字対応へ進める
- Unicode更新時に追従しやすくする

ブランチ候補:

```text
feature/import-unicode-data
```
